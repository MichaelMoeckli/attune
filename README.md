# Attune – User-Centred Personalisation in Fully Automated Radio

Research artefact and pilot-study analysis for the BSc thesis (ZHAW, Institute of Business Information Technology):
**"Attune: User-Centred Personalisation in Fully Automated Radio – A Concept Study in the Context of Digital Wellbeing"**
Author: Michael Möckli · Supervisor: Alexandre de Spindler

Attune is a fully automated, AI-powered radio that uses the same technology stack as engagement-optimised platforms (LLM, TTS, recommender) but inverts the optimisation goal: instead of time-on-platform, it optimises for users' wellbeing (explicit preference input, diversity-oriented curation, a defined session end, transparency).

## Scientific Contribution

The work contributes on three levels:

- **Conceptual** – a theoretically grounded five-building-block model of wellbeing-oriented personalisation, with an explicit mapping of values (Self-Determination Theory, Value-Sensitive Design, algorithmic sovereignty) onto design decisions, condensed into four transferable design principles.
- **Technical** – a working end-to-end artefact that runs the same technology stack as engagement-optimised platforms with an *inverted* objective function and generates personalisation **without collecting any behavioural data**, relying solely on explicit preference input (profiles stored locally in the browser, no tracking, no user account).
- **Empirical** – first, fully reproducible descriptive evidence (open notebook) for the thesis that the well-known platform problems lie in the *objective function*, not in the technology.

## Prototype (`attune-app/`)

Next.js 15 / React 19 / TypeScript. Pipeline: RSS news (SRF, NZZ, Tages-Anzeiger) + OpenWeatherMap → Claude (presenter script) → ElevenLabs (TTS) → MP3 broadcast; optional music via Jamendo or Spotify. Profiles stored locally (localStorage), no user account; in production an invite gate (`/unlock`, password-based) protects access. Every external service has a mock mode.

```bash
cd attune-app
npm install
cp .env.example .env.local   # enter API keys (Anthropic + ElevenLabs required; OpenWeatherMap, Jamendo, Spotify optional)
npm run dev
```

The core of the research idea is the curation logic in `src/services/news.ts` (`selectNews()`): four selection passes **Coverage → Serendipity → Depth → Fallback** operationalise diversity and time-of-day fit as target values — not engagement.

The research-critical selection logic is covered by unit tests (`src/services/news.test.ts`, 11 tests):

```bash
cd attune-app && npm install && npm test
```

## Pilot Study: Validation Across Three Categories

The validation follows the three-way split from Chapter 8 of the thesis; the notebook and HTML export are structured along the same categories:

1. **Feasibility** – Did the pipeline run without errors? Technical quality (audio, intelligibility, voice, transitions, response time). → Notebook Section 2
2. **Usability / Acceptance** – SUS-style usability index (adapted from Brooke 1996), semantic differential, attitude items. → Notebook Section 3
3. **Impact** – Constructs from the theory part (SDT, Digital Wellbeing, algorithmic sovereignty) plus awareness items on filter bubble, bias and limitations. → Notebook Section 4

In addition, Notebook Section 7 answers the four sub-questions of the research question (T1–T4) with one chart each plus a synthesis graphic; Section 8 condenses the findings along the three categories.

## Reproducibility: How the Notebook, HTML and Charts Were Produced

The entire analysis is deterministically reproducible from the raw data file — there are no manually created numbers or charts.

**Procedure:**

1. Raw data export from Google Forms as CSV (`Attune – Pilotstudie.csv`, 96 columns). The questionnaire is included as a PDF (`Attune – Pilotstudie - Google Formulare.pdf`).
2. The notebook loads the CSV, filters for given consent (n = 25), recodes per construct (negatively worded items are reverse-scored, awareness items deliberately are not — rationale in Notebook Section 0.1) and computes descriptive statistics only (M, SD, min, max; no inferential statistics at n = 25).
3. All charts are generated with matplotlib directly in the notebook; the research-question and overview charts are additionally exported as PNG (200 dpi) to `figures/` and embedded into the thesis from there.
4. The HTML export is a 1:1 representation of the executed notebook.

The interactive `Attune_Pilotstudie_Dashboard.html` is a hand-designed presentation layer (Chart.js, standalone). All values shown there — including the exploratory subgroup comparisons and the Spearman correlation matrix — are reproducible from the notebook (Section 10); the notebook remains the authoritative source.

**Run it yourself:**

```bash
pip install pandas numpy matplotlib nbclient nbconvert ipykernel
jupyter nbconvert --to notebook --execute --inplace Attune_Pilotstudie_Auswertung.ipynb
jupyter nbconvert --to html Attune_Pilotstudie_Auswertung.html
```

**Tool transparency:** The notebook and analysis scripts were created with the support of Claude (Anthropic); all coding decisions are documented in the notebook and the outputs were checked against the raw data.

## Key Findings (Summary)

Adapted SUS index 82.5/100 (usability the strongest area); all four sub-questions of the research question above the scale midpoint (T1 autonomy 3.94, T3 session 3.67, T2 diversity and T4 transparency 3.62 each); transparency elements helpful but not consistently noticed; intention to continue use split (14 yes / 11 no). Details: `Attune_Pilotstudie_Auswertung.html` and Chapter 9 of the thesis.
